Game Data Structures

There are three structures (type objects) defined in the fragmon module for holding game data, Overview, Player, and Rule. Overview holds some basic stats for a game server, such as server IP, hostname, team colors and score, etc. Player and Rule describe a single player and rule var, respectively. Fragmon expects multiple Player and Rule objects to be in Python lists. All members listed below can be included in their class's constructor IE, Player(name = “fatbody”, score=55, ping=33)



Overview

Most of the Overview data appears in the query and competition sections of the main Fragmon window. Pay attention to the difference between the type and mode members.

Name of current map

type

string

Type of game (FFA/TEAM/DUAL/etc). This is shown directly to the user without modification.

mode

string

Again this refers to the type of game, but unlike the type member, the mode member is used by Fragmon to determine the game type.

Current values accepted are “1v1” and “team”. In 1v1 mode, the names and scores of the first players with teamid A and teamid B are inserted in place of team names and scores in the competition section.

mod

string

Name of the mod running on the server (OSP, Q4MAX, etc)

string

integer

teamBscore

Score of the second team

teamAcolor

24-bit int

Hex color of the first team (0xff0000)

teamBcolor

24-bit int

Hex color of the second team (0x0000ff)



Player

Data for a single player. Most of this appears in the player list in the main Fragmon window.

Member

Type

Description

name

string

Name of the player. This shouldn't have any color information.

score

int

Player's score

ping

int

Player's ping

team

string

Player's team name

teamid

int

The team the player is playing in: 0 is team A, 1 team B, and anything else is team F (specs/other)

id

int

Player's connection ID. Fragmon uses this to match players when the user overrides teams in the player list
